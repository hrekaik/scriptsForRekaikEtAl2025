# ATAC-seq

## Fastq to peaks/coverage

This analysis was computed using a local [galaxy](https://doi.org/10.1093/nar/gkac247) server.

The workflow used has been exported [here](./Galaxy-Workflow-ATACseq.ga). It has been run with the following inputs:

- PE fastq input: will be available on SRA/GEO
- reference_genome: mm10
- effective_genome_size: 1870000000
- bin_size: 50

The command lines used have been written [here](./command_lines.sh).

## Figure generation

The coverages were generated by [pyGenomeTracks](https://pygenometracks.readthedocs.io/en/latest/).

With the following commands:

```bash
mkdir -p outputs/ATACseq/
pyGenomeTracks --tracks ATACseq/ES_ATAC.ini  --region chr6:52136234-52287891 --outFileName outputs/ATACseq/ATAC_HoxA.pdf

pyGenomeTracks --tracks ATACseq/ES_ATAC.ini  --region chr11:96175975-96388358 --outFileName outputs/ATACseq/ATAC_HoxB.pdf

pyGenomeTracks --tracks ATACseq/ES_ATAC.ini  --region chr15:102902350-103068692 --outFileName outputs/ATACseq/ATAC_HoxC.pdf

pyGenomeTracks --tracks ATACseq/ES_ATAC.ini  --region chr2:74635504-74790737 --outFileName outputs/ATACseq/ATAC_HoxD.pdf
```

The output images can be found [here](../outputs/ATACseq/).

To show that both replicates were identical here are the commands to obtain figures that were not used in the paper:

```bash
mkdir -p outputs/ATACseq/
pyGenomeTracks --tracks ATACseq/ES_ATAC_all_reps.ini  --region chr6:52136234-52287891 --outFileName outputs/ATACseq/ATAC_HoxA_all_reps.pdf

pyGenomeTracks --tracks ATACseq/ES_ATAC_all_reps.ini  --region chr11:96175975-96388358 --outFileName outputs/ATACseq/ATAC_HoxB_all_reps.pdf

```
